# CDP Setup

Local development for the CDP monorepo (`scrapers/` + `muvstok-api/` + `n8n/`).

## Prerequisites

- Docker (Postgres + Redis)
- Python 3.12+, `uv` (muvstok-api)
- Node 18+ (optional — n8n workflow SDK under `n8n/sdk/`, generated by `make sync-n8n`)

## Quick start (monorepo root)

```bash
# Shared infra
make dev

# Scraper API + worker (separate terminals)
make -C scrapers setup
make -C scrapers dev      # API :8000
make -C scrapers worker   # Celery

# StokAPI
cd muvstok-api && docker compose -f docker/docker-compose.yml up -d
cd muvstok-api && uv sync && uv run uvicorn app.main:app --reload
# Worker: see muvstok-api/README.md
```

## Environment

- Copy service `.env.example` files; never commit secrets.
- Production secrets: Azure Key Vault `cdp-scrapers-kv-prod`.
- Scraper cache: Redis DB 1 + `SCRAPE_CACHE_*` vars (see `scrapers/docs/SCRAPE_CACHE_OPERATIONS.md`).

## n8n (local edit only)

Router Code: `n8n/src/*.js` → inject → optional publish:

```bash
python3 scripts/sync_workflow_code_from_shared.py
make sync-n8n   # requires explicit user approval for live publish
```

## Verify

```bash
make -C scrapers test lint
make check-muvstok
bash scripts/smoke_dual_pipeline.sh   # when APIs + n8n are reachable
```
