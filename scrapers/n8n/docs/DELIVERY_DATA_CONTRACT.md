# Delivery And Data Contract

> **Canonical:** [docs/n8n/DATA_CONTRACTS.md](../../../docs/n8n/DATA_CONTRACTS.md) and [contracts/scraper-callback.schema.json](../../../contracts/scraper-callback.schema.json).

Last updated: 2026-05-14

## What We Deliver

The service delivers structured automotive part lookup data. For each searched
SKU, it searches selected supplier/marketplace sites, normalizes the results,
persists job data in PostgreSQL, and returns JSON that downstream automation can
store, compare, and review.

The operational answer is:

```text
For this exact SKU, which visible source listings have trustworthy price,
currency, condition, availability, seller, URL, origin, and raw source title?
```

## Result Hierarchy

The API returns a nested tree:

```text
ScrapeJobResult
  results[]             one per requested SKU
    site_results[]      one per source site
      results[]         one per product/listing/offer found on that site
```

N8N destinations such as Google Sheets usually need this flattened into one row
per product result, plus placeholder rows for checked-but-missing site results.

## SKUResult Fields

| Field | Meaning |
|---|---|
| `sku` | SKU requested by the client. |
| `brand` | Brand from request, possibly blank. |
| `site_results` | Per-site status and listings. |
| `best_price` | Lowest exact-match priced result when all priced exact matches share one currency. |
| `total_results` | Count of product/listing records returned across all searched sites. |

## SiteResult Fields

| Field | Meaning |
|---|---|
| `site` | Site ID such as `gm`, `ml`, `vw`, `eu`, `pecadireta`, `melibox`. |
| `site_name` | Human-readable source name. |
| `status` | Source-level outcome. |
| `error_message` | Details when status is `error`, `blocked`, or `timeout`. |
| `results` | Array of `PartResult` records. Empty when not found or blocked. |
| `search_time_ms` | Time spent on that site for this SKU. |

Site statuses:

| Status | Meaning | How N8N should treat it |
|---|---|---|
| `success` | Source returned one or more usable result records. | Write result rows and include in summaries. |
| `not_found` | Search completed but no exact product was found. | Write a placeholder row if operations needs audit coverage. |
| `no_price` | Product evidence exists but no positive price is visible. | Write placeholder or diagnostic row; not a priced quote. |
| `blocked` | Source blocked the scraper, CAPTCHA/access-denied/403. | Alert or track source health. Do not treat as no inventory. |
| `timeout` | Site did not complete within timeout. | Retry later or route to manual review. |
| `error` | Unexpected scraper/application exception. | Alert with `error_message`. |

## PartResult Fields

| Field | Meaning |
|---|---|
| `sku_searched` | Requested SKU used for the search. |
| `sku_found` | SKU evidence found on the source. |
| `exact_match` | Whether `sku_found` or source evidence matches the normalized searched SKU. |
| `site` | Site ID. |
| `site_name` | Human source name. |
| `price` | Numeric price, or `null` when no price. |
| `currency` | `BRL`, `USD`, or `EUR`. |
| `condition` | `new`, `used`, or `unknown`. |
| `availability` | Source-visible stock/status text. |
| `seller_name` | Dealer/seller/supplier name when visible. |
| `product_url` | Direct product/listing URL when available. |
| `origin` | Region/source origin, such as `Brasil`, `Europa`, `EUA`, `China`. |
| `scraped_at` | ISO timestamp generated by the scraper. |
| `raw_title` | Original visible listing/product title. |

## Best Price Rule

The API chooses `best_price` only from results where:

```text
price is positive
exact_match is true
all priced exact matches use the same currency
```

If exact matches include mixed currencies, `best_price` is `null`. N8N agents
must not compare `BRL` against `USD` or `EUR` unless a separate currency
conversion step is explicitly added.

## Recommended Flat Output Columns

Use these columns for Google Sheets, Airtable, databases, or CSV exports:

```text
job_id
job_status
sku
brand
site
site_name
site_status
site_error_message
sku_searched
sku_found
exact_match
price
currency
condition
availability
seller_name
product_url
origin
raw_title
scraped_at
search_time_ms
is_best_price
job_started_at
job_completed_at
job_duration_seconds
```

For site statuses with no `PartResult`, create one placeholder row with:

```text
sku_searched = SKUResult.sku
sku_found = ""
exact_match = false
price = empty
currency = ""
availability = site_status
raw_title = site_status uppercased, for example NOT_FOUND or BLOCKED
```
