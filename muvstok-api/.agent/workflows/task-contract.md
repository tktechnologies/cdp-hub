# Task Contract

Every task should define:

- task ID or short name
- goal
- files allowed to change
- acceptance criteria
- API contract changes
- database changes
- Azure infrastructure impact
- required tests or Azure validation
- required logs, metrics, and traces
- required docs and `.agent/` updates
- risks
- done/not-done checklist

Use this contract before large implementation work, production-facing behavior changes, or cloud validation work.
